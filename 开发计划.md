# 课程表安卓应用 — 开发计划

## 技术栈
| 层级 | 选型 |
|---|---|
| UI | Jetpack Compose + Material Design 3 |
| 架构 | MVVM + Repository 模式 |
| 依赖注入 | Hilt |
| 本地存储 | Room + DataStore |
| 后台任务 | AlarmManager（精确提醒）+ WorkManager（保活/重调度）|
| 通知 | NotificationManager + 各厂商实时活动 SDK |
| 语音 | Android TTS（TextToSpeech）|
| 构建 | Gradle + Kotlin DSL |
| 教务解析 | Jsoup（HTML 解析）+ 可扩展 Parser 架构 |
| 文件解析 | Apache POI（Excel）、OpenCSV（CSV）、iCal4j（ICS）|
| 图片 OCR | ML Kit（Google 官方，离线识别）|
| 网络请求 | Retrofit2 + OkHttp + Gson |
| 图片加载 | Coil（Compose 首选）|
| 异常监控 | Firebase Crashlytics |
| 远程配置 | Firebase Remote Config（热更新开关/解析器配置）|

## 阶段二点五：课程导入系统（第 3 周，与阶段二并行启动）

### 2.5.1 文件导入
- [ ] Excel (.xlsx) 导入：Apache POI 解析，支持课程名/教室/教师/时间/周数字段识别
- [ ] CSV 导入：OpenCSV 解析，支持表头自动匹配
- [ ] ICS (iCalendar) 导入：iCal4j 解析，支持 RRULE 周循环展开
- [ ] 导入预览页：解析结果展示 + 冲突检测 + 选择性导入
- [ ] 导入模板下载（引导用户按模板格式整理数据）

### 2.5.3 分享与快捷导入（参考 WakeUp 课程表）
- [ ] **分享码/口令导入**：生成短分享码，他人输入即可导入相同课表
- [ ] **图片导入**：ML Kit OCR 识别课表截图/照片，解析为课程数据
  - [ ] 截图区域智能识别（自动框选课表区域）
  - [ ] OCR 结果后处理：文本清洗、时间格式识别、字段匹配
  - [ ] 识别结果预览与手动校正
- [ ] **每日自动刷新**：WorkManager 定时任务，后台同步教务系统最新课表
- [ ] **课表分享**：导出为图片/文件/分享码多种方式分享给同学

### 2.5.4 教务系统导入
- [ ] 内置 WebView：加载教务系统登录页，拦截 Cookie / 自动提交表单
- [ ] 课程表页面抓取：用户登录后自动跳转课表页，注入 JS 提取 HTML
- [ ] 预置解析器实现（至少 3 种）：
  - [ ] 强智教务系统
  - [ ] 正方教务系统
  - [ ] 青果教务系统
- [ ] Parser 插件化架构：抽象 `CourseParser` 接口，新学校适配仅需实现接口
- [ ] HTML 源码粘贴模式：用户粘贴课表页面源码，离线解析
- [ ] 可视化字段映射：解析失败时让用户拖拽确认字段对应关系
- [ ] 登录状态管理：Cookie 持久化，支持会话过期重新登录

## 阶段三：提醒与语音系统（第 4-5 周）

- [ ] AlarmManager 封装（设置/取消/更新课前提醒）
- [ ] 开机广播接收器（BOOT_COMPLETED）重调度所有提醒
- [ ] 时区变更、日期变更广播处理
- [ ] WorkManager 每日定时任务：预生成第二天的 Alarm
- [ ] Doze 模式测试与适配（setExactAndAllowWhileIdle）

### 3.2 通知系统
- [ ] 基础通知渠道创建（重要提醒 / 普通提醒 / 日程提醒）
- [ ] 课程通知：课程名、教室、倒计时
- [ ] **日程通知**：考试/作业截止提醒，自定义事件提醒
- [ ] **每日概览推送**：每天固定时间推送次日课程与待办摘要（参考 WakeUp 课程表）
- [ ] 通知动作："我知道了" / "延后 5 分钟再提醒" / "标记完成"（作业/任务）

### 3.3 语音播报
- [ ] TextToSpeech 初始化与生命周期管理
- [ ] 播报文案模板引擎（支持变量：{courseName}, {classroom}, {minutes}）
- [ ] 播报触发逻辑（绑定在 AlarmReceiver 中）
- [ ] 静音/振动模式下的音频播放（使用 AudioManager 申请焦点）
- [ ] 用户可配置：语音开关、播报语速、音量

### 3.4 上下课倒计时与实时状态
- [ ] **上下课倒计时**：首页/Widget/通知栏实时显示距离上课/下课还有多久
- [ ] **正在上课状态检测**：根据当前时间自动高亮正在上的课程
- [ ] **下课提醒**：课程结束后推送通知，提示下一节课信息

### 3.6 上课自动静音（参考云舒课程表）
- [ ] AudioManager 封装：上课开始时切换为振动/静音模式
- [ ] 下课自动恢复原始铃声模式
- [ ] 用户可配置：静音模式（完全静音 / 振动）/ 豁免课程（体育课等不需要静音的课程）
- [ ] 与 AlarmManager 联动，精确触发静音/恢复

### 3.7 系统日历同步
- [ ] Calendar Provider API 封装：将课程/考试/作业写入系统日历
- [ ] 双向同步开关：用户可选择是否同步到系统日历
- [ ] 同步冲突处理：课程修改后更新系统日历事件

## 风险与应对

| 风险 | 影响 | 应对策略 |
|---|---|---|
| 各厂商灵动岛 API 不开放或限制多 | 高 | 优先实现华为实况窗 + 悬浮窗 fallback，不阻塞主线 |
| AlarmManager 在国产 ROM 被杀后台 | 高 | 引导用户开启自启动/电池白名单；使用前台 Service 保活 |
| TTS 引擎缺失（部分精简系统） | 中 | 首次启动检测 TTS 可用性，引导用户安装；提供纯通知 fallback |
| 教务系统登录需要验证码 / 二次认证 | 高 | WebView 内用户手动完成验证；提供 HTML 源码粘贴作为备选方案 |
| 不同学校教务系统 HTML 结构差异大 | 中 | Parser 插件化架构 + 社区贡献适配器；可视化字段映射兜底 |
| 系统日历同步需要日历读写权限 | 低 | 首次使用时动态申请权限；用户拒绝时提供手动导出 ICS 备选方案 |
| OCR 识别课表截图准确率不稳定 | 中 | ML Kit 为主，提供手动校正界面；复杂截图提示用户使用文件导入 |
| 教务系统反爬虫 / IP 封禁 | 中 | 控制请求频率；使用校园网优先策略；异常时引导用户手动导入 |
| 成绩查询涉及敏感隐私数据 | 中 | 本地存储加密（EncryptedSharedPreferences）；不上传用户成绩到第三方服务器 |
